STM32F411 智能家居环境监测系统

传感器采集 · TFT 显示 · ESP8266 Wi-Fi · MQTT 云端 · PCB 打样

STM32F411 系统实物运行 · DHT11 / DS18B20 传感器采集与 TFT 显示

本项目基于 STM32F411CEU6 设计并实现了一套智能家居环境监测系统，集成 DHT11、DS18B20、ST7789V TFT 彩屏、ESP8266 Wi-Fi、MQTT 云端上传、按键菜单、ADC 采集和蜂鸣器报警等功能。项目覆盖原理图设计、PCB 打样、焊接调试、外设驱动开发、显示界面设计、无线通信和云端验证，体现了从硬件到软件的完整嵌入式系统实现流程。

我的主要工作

本项目由我完成整体方案设计、硬件选型、原理图与 PCB 设计、焊接调试、STM32 固件开发、外设驱动封装、TFT 菜单界面、ESP8266 联网通信和 MQTT 云端验证。项目重点不是简单拼接模块，而是完成从硬件设计到软件调试的完整嵌入式系统闭环。

  • 使用 STM32F411CEU6 作为主控，完成 GPIO、SPI、DMA、UART、TIM、ADC 等外设配置
  • 基于 SPI + DMA 驱动 ST7789V TFT 彩屏，设计局部刷新策略，降低刷屏闪烁
  • 编写 DHT11、DS18B20 传感器采集逻辑，其中 DS18B20 使用非阻塞状态机避免主循环阻塞
  • 使用 ESP8266 通过 UART AT 指令连接 Wi-Fi，并将环境数据通过 MQTT 上传到云端

  • 使用串口助手、MQTTX、万用表等工具验证系统运行状态

系统架构

传感器层 DHT11 DS18B20 ADC → STM32F411CEU6
数据采集与处理 → TFT 菜单显示 蜂鸣器报警 通信层 ESP8266
UART AT · Wi-Fi → MQTT 云端上传 → MQTTX 验证

关键难点

1. SPI + DMA 驱动 ST7789V TFT

ST7789V 彩屏分辨率为 320×480，如果使用普通阻塞式 SPI 全屏刷新，容易出现刷新慢、界面闪烁和主循环被占用的问题。

我使用 SPI + DMA 方式传输显示数据，将 CPU 从大量像素发送中释放出来，并设计局部刷新策略，只更新发生变化的区域，减少无效刷屏。最终实现 TFT 菜单界面流畅显示，屏幕刷新率可达 50+ fps，DMA 优化后清屏时间小于 1ms。

2. DS18B20 非阻塞状态机采集

DS18B20 温度转换时间较长，典型转换时间约 750ms。如果直接在主循环中阻塞等待，会影响 TFT 刷新、按键响应和 MQTT 上传。

因此我将 DS18B20 采集流程拆分为状态机：启动转换、等待转换、读取数据、数据处理。主循环不再等待传感器转换完成，而是在不同时间片中轮询状态，从而保证系统界面和通信任务可以继续运行。

typedef enum { DS18B20_IDLE, DS18B20_START_CONVERT, DS18B20_WAIT_CONVERT, DS18B20_READ_READY } ds18b20_state_t;

3. ESP8266 + MQTT 分时上传

系统通过 ESP8266 使用 UART AT 指令连接 Wi-Fi，并将传感器数据上传到 MQTT 云端。调试过程中发现，如果多个传感器数据在短时间内连续上传，可能导致 ESP8266 串口缓冲区压力增大，出现响应延迟或数据丢失。

为此我采用分时上传策略，将温度、湿度、ADC 等不同数据分散到不同时间片上传，降低瞬时通信压力。同时通过串口日志和 MQTTX 验证云端消息是否成功接收。

4. PCB 打样后的供电干扰复位问题

PCB 打样和焊接完成后，多模块同时工作时出现过系统复位或运行不稳定现象。问题主要与供电瞬态波动、模块启动电流和电源走线有关。

我通过增加去耦电容、优化供电回路、检查焊接连接、加宽关键电源走线，并在软件上采用分时启动策略，降低多个模块同时启动造成的电源冲击。最终系统能够稳定运行。

PCB 设计要点

  • 每个 STM32 VDD 引脚附近放置 100nF 去耦电容，并在电源入口增加较大容量滤波电容
  • 对 TFT、ESP8266 等电流变化较大的模块，优化供电走线，减少压降和干扰
  • SPI、UART 等高速或关键通信线尽量缩短路径，避免靠近强干扰区域
  • 预留调试接口，方便串口日志输出、固件下载和后期问题定位
  • 对按键、蜂鸣器、传感器接口进行模块化布局，方便焊接和调试

调试与验证

本项目通过多种方式验证系统功能：

  • 使用串口助手输出系统运行状态、ESP8266 AT 指令响应和 MQTT 上传结果
  • 使用 MQTTX 验证云端是否成功接收到传感器数据
  • 通过 TFT 屏幕实时显示温湿度、菜单状态和报警信息
  • 通过按键长短按测试菜单切换和交互逻辑
  • 通过上电运行测试验证 PCB 焊接、供电和模块稳定性

项目复盘

这个项目让我更加理解了嵌入式系统不是单个外设驱动的简单叠加，而是硬件、驱动、任务调度、通信和电源稳定性共同配合的整体工程。TFT 显示、传感器采集、MQTT 上传和按键交互都运行在同一个主循环中，因此必须通过状态机、分时调度和 DMA 降低阻塞，保证系统响应性。

在硬件调试方面，PCB 能画出来并不代表系统一定能稳定运行。ESP8266、TFT 屏幕和传感器同时工作时，供电稳定性、去耦电容布局、电源走线和模块启动顺序都会影响整机可靠性。这个项目让我建立了从现象观察、串口日志、硬件测量到软件调整的完整排查思路。
