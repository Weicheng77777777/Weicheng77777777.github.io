STM32F411 智能家居环境监测系统

传感器采集 · TFT 显示 · ESP8266 Wi-Fi · MQTT 云端 · PCB 打样

▷ 查看 GitHub 仓库

项目背景

本项目是一个基于 STM32F411CEU6 的智能家居环境监测系统，集成 DHT11 温湿度传感器、 DS18B20 温度传感器、ST7789V 3.5 寸 TFT 彩屏、ESP8266 Wi-Fi 模块、MQTT 云端上传、 按键菜单交互、ADC 模拟采集和蜂鸣器提示等功能。项目覆盖原理图设计、PCB 制版、焊接调试、 外设驱动开发、菜单界面设计、无线通信和云端数据上传，完整体现嵌入式系统工程化实现流程。

硬件组成

  • 主控：STM32F411CEU6 (Cortex-M4, 100MHz)
  • 显示：ST7789V 3.5 寸 TFT 彩屏 (320×480, SPI + DMA)
  • 传感器：DHT11 温湿度 / DS18B20 温度
  • 通信：ESP8266 (UART2 + AT 指令) → Wi-Fi → MQTT
  • 交互：按键 (长短按识别) + 蜂鸣器

关键难点与解决方案

1. SPI + DMA 驱动 ST7789V TFT

使用 SPI + DMA 实现高速刷屏，设计局部刷新和缓冲显示策略减少闪烁。 实测屏幕刷新率可达 50+ fps，DMA 优化后清屏时间小于 1ms。

2. DS18B20 非阻塞状态机采集

DS18B20 转换时间约 750ms，采用非阻塞状态机设计避免阻塞主循环：

typedef enum { DS18B20_IDLE, DS18B20_START_CONVERT, DS18B20_WAIT_CONVERT, DS18B20_READ_READY } ds18b20_state_t;

3. MQTT 分时上传防丢包

MQTT 消息过密可能导致丢包，采用分时上传策略，将不同传感器数据分散到不同时间片上传， 避免 ESP8266 缓冲区溢出。

4. 供电干扰复位问题

多模块同时工作导致供电干扰复位。通过增加去耦电容、优化电源走线和分时启动策略解决。

PCB 设计要点

  • 去耦电容布局：每个 VDD 引脚就近放置 100nF + 10μF
  • 供电回路优化：加宽电源走线，减少压降
  • 信号完整性：SPI 走线等长，远离高频干扰源

复盘总结

这个项目让我深刻理解了：

  • 状态机设计 — 非阻塞状态机是嵌入式系统任务调度的基础
  • 模块化驱动 — 每个外设独立封装，降低耦合度，提高可维护性
  • 供电稳定性的重要性 — 电路能画出来不等于板子能稳定工作
  • 分时复用 — 多任务共享总线时，时间片分配和优先级管理至关重要
