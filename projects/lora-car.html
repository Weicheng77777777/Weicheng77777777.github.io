LoRa 双端遥控履带车系统

双 MCU · LoRa 双向通信 · PWM 电机控制 · GPS 解析 · 状态机调度

比赛实物小车 · 履带底盘 + 车端控制板 + LoRa + GPS + 电机驱动模块

本项目是一个基于 STM32F1 控制端与 STM32F4 车端的双 MCU 无线遥控履带车系统。控制端负责按键输入、OLED 状态显示、控制指令发送和回传数据解析；车端负责 PWM 电机控制、GPS 数据采集、ADC 模拟量采集和 LoRa 数据回传。系统通过 LoRa 模块实现双向无线通信，并采用自定义数据包、校验机制和状态机调度，提高通信可靠性和实时响应能力。

  • 基于 STM32F4 实现车端控制逻辑，负责电机 PWM 输出、传感器采集和 LoRa 数据回传
  • 基于 STM32F1 实现控制端逻辑，负责按键输入、OLED 显示、指令发送和回传数据解析
  • 使用定时器 PWM 控制电机驱动，实现前进、后退、转向、停止等运动动作

  • 使用超级循环 + 状态机方式组织任务，避免电机控制、通信接收、GPS 解析和显示刷新相互阻塞

关键难点

1. LoRa 双向通信与自定义协议

LoRa 通信距离较远，但速率较低，并且在实际环境中可能出现丢包、半包、异常字符或数据错位。如果直接发送裸数据，控制端和车端很难判断一帧数据是否完整。

因此我设计了简单的数据包格式，在每帧数据中加入帧头、命令类型、数据负载、校验字段和帧尾。接收端通过状态机逐字节解析，只有帧头、长度、校验和帧尾都符合要求时才执行指令或更新数据。

/* 自定义数据包格式 */ typedef struct { uint8_t header; // 帧头 0xAA uint8_t cmd; // 命令类型 uint8_t data[8]; // 数据负载 uint8_t checksum; // 校验 uint8_t footer; // 帧尾 0x55 } lora_packet_t;

这不是简单的 UART 透传——通过帧格式和校验机制，可以有效减少误动作风险，体现的是嵌入式系统中通信协议设计能力。

2. PWM 电机控制与运动逻辑

车端需要根据控制端下发的指令实时控制电机，实现前进、后退、左转、右转和停止等动作。电机控制不仅要输出 PWM 占空比，还要处理方向控制、速度变化和动作切换。

我使用 STM32 定时器 PWM 输出控制 DRV8701 电机驱动模块，通过调整占空比控制速度，通过 GPIO 控制方向信号。在主循环中，车端根据 LoRa 解析出的命令更新运动状态，而不是在通信接收函数中直接执行长时间阻塞动作，从而保证通信和运动控制可以持续响应。

3. GPS NMEA 数据解析

GPS 模块通过串口持续输出 NMEA 0183 格式数据，其中常见语句包括 $GPGGA 和 $GPRMC。实际串口数据是连续字节流，可能出现半包、断帧或无效定位状态，因此不能只依赖简单字符串截取。

我在串口接收逻辑中按字符缓存 GPS 数据，并通过状态机判断一条 NMEA 语句是否接收完整。随后解析经纬度、时间和定位状态等字段，只在数据完整且定位有效时更新车端状态，并通过 LoRa 回传给控制端。

4. 超级循环 + 状态机非阻塞调度

这个项目同时包含电机控制、LoRa 收发、GPS 解析、ADC 采集、OLED 显示和按键扫描。如果某个任务使用长时间 delay 或阻塞等待，就会导致遥控响应变慢、通信丢包或显示刷新不及时。

因此我采用超级循环 + 状态机的方式组织任务。每个任务只在自己的时间片内完成短时间处理，避免单个任务长期占用 CPU，提高系统实时性和稳定性。

按键扫描→ LoRa 接收→ 指令解析→ 电机状态更新→ GPS 解析→ ADC 采集→ OLED 显示→ 状态回传

这是裸机系统中的任务调度思想——不是简单 while 循环堆代码，而是通过状态机让多任务协调运行，避免相互阻塞。

项目复盘

这个项目让我理解到，无线遥控小车的难点不只是让电机转起来，而是让控制端、通信链路和车端执行逻辑稳定配合。LoRa 通信在实际环境中可能出现延迟、丢包或异常字节，因此必须通过帧格式、校验和接收状态机保证指令不会被误解析。

在裸机 STM32 系统中，多模块同时运行时最容易出现的问题是阻塞。GPS 串口接收、LoRa 通信、电机控制、ADC 采样和 OLED 刷新如果都直接使用 delay 或阻塞等待，系统响应会明显变慢。因此我在项目中使用超级循环 + 状态机拆分任务，让每个任务短时间执行，提高系统实时性。

这个项目也让我积累了整机联调经验。实际小车运行时，供电、电机干扰、串口通信、模块接线和数据格式都会影响最终效果。相比单独点亮某个模块，把多个模块组合成一个能跑、能通信、能回传数据的系统，更能体现嵌入式工程能力。
