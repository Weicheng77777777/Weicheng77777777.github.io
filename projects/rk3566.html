RK3566 泰山派 Linux 训练营

Linux BSP 入门 · 设备树修改 · 内核编译 · 显示/触摸调试

项目背景

基于 RK3566 平台进行嵌入式 Linux 入门实践，围绕 Linux 开发环境搭建、设备树修改、 内核编译、镜像烧录、显示接口调试和触摸模块调试开展训练，建立对 Linux BSP 和驱动适配流程的基础认知。

主要工作

  • 完成 RK3566 Linux 开发环境搭建，使用 Linux 命令行进行设备树修改、内核编译和镜像烧录
  • 理解 Kernel、DTB、rootfs 和启动镜像之间的配合关系
  • 调试 HDMI、eDP 30pin、MIPI 等显示接口，理解不同显示链路在设备树配置、接口连接上的差异
  • 完成 GT9271 电容触控模块 I2C 调试，围绕 I2C 地址、设备树节点、驱动匹配和事件验证进行问题定位
  • 形成"修改配置 → 编译烧录 → 上板验证 → 日志分析 → 继续修正"的 Linux BSP 调试闭环

调试工具

  • 串口调试助手（查看启动日志）
  • dmesg（内核日志分析）
  • /proc/device-tree（验证设备树节点）
  • /sys/bus/i2c/devices（I2C 设备扫描）

最终结果

✅ Linux 开发环境搭建完成
✅ 内核编译与镜像烧录成功
✅ HDMI/eDP/MIPI 显示接口调试
✅ GT9271 触摸功能正常

复盘总结

  • Linux BSP 调试的核心流程是 "改 → 编 → 烧 → 测 → 看日志 → 再改"
  • 设备树修改后必须重新编译 DTB 并确认烧录到正确分区
  • 不同显示链路（HDMI/eDP/MIPI）的设备树配置差异很大，需要对照 datasheet 逐个确认
